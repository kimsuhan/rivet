# 공통 설정 패키지 작업 지침

## 적용 범위

- 이 지침은 `packages/config` 아래의 모든 파일에 적용한다.

## 필수 문서

- 코드를 구현, 리뷰 또는 리팩터링하기 전에 [공통 개발 지침](../../docs/development/common.md)을 읽는다.
- 특정 앱 전용 설정을 변경할 때는 해당 앱의 개발 지침과 `AGENTS.md`를 추가로 읽는다.

## 책임 경계

- 공통 TypeScript, ESLint와 Prettier 설정만 소유한다.
- 런타임 제품 규칙, 환경별 비밀 값과 앱 전용 설정을 공통 패키지에 넣지 않는다.
- 모든 앱에 필요하지 않은 plugin이나 rule을 공통 의존성으로 강제하지 않는다.
- 소비자가 실제 사용하는 공개 preset만 export하고 미래용 설정을 미리 만들지 않는다.

## 완료 조건

- 변경한 preset을 사용하는 실제 앱이나 패키지에서 lint와 typecheck를 실행한다.
- 포맷 설정 변경은 대표 파일에 적용해 불필요한 전체 저장소 diff가 생기지 않는지 확인한다.
- 설정 package의 공개 export와 각 소비자 참조가 일치하는지 확인한다.
