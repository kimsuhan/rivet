# 이벤트 계약 패키지 작업 지침

## 적용 범위

- 이 지침은 `packages/event-contracts` 아래의 모든 파일에 적용한다.

## 필수 문서

- 구현·리뷰 전에 [공통 개발 지침](../../docs/development/common.md)을 읽는다.
- 이벤트 동작이 필요할 때만 [비동기 작업 명세서](../../docs/architecture/005.%20실시간%C2%B7알림%20및%20비동기%20작업%20명세서.md)의 해당 이벤트를 읽는다.

## 책임 경계

- 실제 API 생산자와 Worker 소비자가 공유하는 `eventType`, `schemaVersion`, 최소 payload와 순수 검증만 공개한다.
- 데이터베이스 모델, 앱 서비스, HTTP DTO, 로그와 외부 공급자 코드를 넣지 않는다.
- 이메일, 토큰 원문, 업무 본문과 사용자 표시 값을 이벤트 payload에 추가하지 않는다.
- 새 이벤트는 실제 생산자와 소비자가 같은 변경에서 생길 때만 추가한다.

## 호환성

- 배포 중 이전 생산자와 새 소비자가 공존할 수 있도록 기존 버전을 임의로 변경하지 않는다.
- 호환되지 않는 payload 변경은 `schemaVersion`을 올리고 소비자 전환 순서를 함께 검증한다.
- 알 수 없는 버전과 불필요한 필드는 검증에서 명시적으로 거부한다.

## 완료 조건

- 공개 계약의 정상·잘못된 payload·지원하지 않는 버전을 단위 테스트한다.
- lint, typecheck, test와 build를 실행한다.
