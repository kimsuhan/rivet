# A4 독립 검토 프롬프트

> 사용자 열람·복사용 문서다. `009` 구현 세션과 분리된 새 세션에서 실행한다.

## 실행 설정

| 항목 | 추천 |
| --- | --- |
| 모델 | Claude Sonnet |
| 추론 강도 | high |

```text
Rivet Alpha A4 이슈 템플릿 구현을 독립적으로 검토해줘. 이번 세션은 읽기 전용 검토이며 코드와 문서를 수정하지 마.

009 구현 세션의 최종 보고, Alpha 템플릿 요구사항, A4 실행 계획, 이슈 생성·프로젝트 역할 계약, 실제 diff와 테스트를 대조해.

다음을 확인해:
- 관리자 관리 권한과 일반 멤버 활성 목록 조회 경계
- 설명 Markdown, 기본 속성, 보관과 version 저장 계약
- 적용 시 현재 라벨·프로젝트·역할·팀 권한 재검증
- 기존 입력 덮어쓰기 확인과 적용 뒤 자유로운 수정
- 템플릿 변경이 과거 이슈에 영향을 주지 않는지
- 보관·권한 상실·오래된 version과 동시 수정 처리
- 활동·분석 이벤트에 템플릿 본문 전체가 복제되지 않는지
- 생성 이슈와 템플릿 사이 지속 동기화가 생기지 않았는지
- 설정·생성 UI의 키보드 접근성과 실패 시 입력 복구
- API, 생성 클라이언트, 정본 문서와 테스트 정합성

결과는 심각도 순 findings, 근거, 누락된 테스트와 최종 판정으로 작성해. 판정은 PASS 또는 CHANGES_REQUIRED 중 하나만 사용해. CHANGES_REQUIRED이면 009 구현 세션으로 돌려보낼 수정 목록을 붙여. commit, push, deploy는 하지 마.
```
