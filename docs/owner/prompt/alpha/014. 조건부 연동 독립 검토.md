# 조건부 연동 독립 검토 프롬프트

> 사용자 열람·복사용 문서다. `013` 판단 세션과 분리된 새 세션에서 실행한다.

## 실행 설정

| 항목 | 추천 |
| --- | --- |
| 모델 | Claude Opus |
| 추론 강도 | high |

```text
Rivet Alpha의 GitHub·Slack 조건부 연동 판단 결과를 독립적으로 검토해줘. 이번 세션은 읽기 전용 검토이며 코드와 정본 문서를 수정하지 마.

013 판단 세션의 보고서, Alpha 제품 범위의 조건부 연동 기준, A5에서 확보된 실제 피드백·제품 이벤트 근거를 대조해. GitHub, Slack, 둘 다 미구현의 세 선택지를 같은 기준으로 다시 평가해.

다음을 확인해:
- 반복 문제를 뒷받침하는 workspace·사용자 수와 빈도 근거가 실제로 있는지
- 현재 우회 비용과 첫 범위로 줄어드는 조작이 과장되지 않았는지
- 외부 권한·비밀·webhook·rate limit·장애 대응 비용이 반영됐는지
- 보존·삭제·감사와 공급자 장애 영향이 빠지지 않았는지
- 구현하지 않을 때 Alpha·Beta 영향이 구체적인지
- GitHub·Slack 동시 구현이나 양방향 자동화로 범위가 확장되지 않았는지
- 근거 부족 시 미구현을 선택한다는 원칙을 지켰는지

결과는 013 결론에 대한 동의 또는 반대, 근거 차이, 불확실성, 추천 선택과 최종 판정으로 작성해. 판정은 CONFIRMED 또는 RECONSIDER 중 하나만 사용해. RECONSIDER이면 013 세션에서 다시 판단할 질문과 필요한 증거를 붙여. commit, push, deploy는 하지 마.
```
