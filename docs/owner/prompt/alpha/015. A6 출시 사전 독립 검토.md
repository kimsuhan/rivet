# A6 출시 사전 독립 검토 프롬프트

> 사용자 열람·복사용 문서다. `016` 최종 출시 판정 전에 새 세션에서 실행한다.

## 실행 설정

| 항목 | 추천 |
| --- | --- |
| 모델 | Claude Opus |
| 추론 강도 | high |

```text
Rivet Closed Alpha 최종 출시 판정 전에 전체 상태를 독립적으로 사전 검토해줘. 이번 세션은 검토 전용이며 코드와 문서를 수정하지 마.

A0~A5 구현·독립 검토 결과, 조건부 연동의 승인된 결론, 현재 작업 트리, Alpha 제품 범위·실행 계획, 테스트·배포 및 운영 명세와 배포·복구 가이드를 대조해. 필요한 로컬 읽기·검증 명령은 실행할 수 있지만 운영 배포, 외부 초대, DNS·TLS·비밀 변경은 하지 마.

다음을 확인해:
- 각 마일스톤 독립 검토가 PASS이고 미해결 finding이 없는지
- 조건부 연동 결정이 CONFIRMED이며 선택된 구현이 있다면 완료·검증됐는지
- migration, OpenAPI, 생성 클라이언트와 문서 계약 불일치가 없는지
- workspace 경계, 부분 저장, 중복 Push·이벤트와 민감 정보 노출 위험
- 핵심 desktop·compact·mobile 사용자 흐름과 실패 복구 증거
- 전체 lint, typecheck, unit, integration, contract, build와 E2E를 막을 알려진 문제가 없는지
- 백업·복구, 관측, 운영 인증 이메일과 Push 운영 설정의 남은 승인 항목
- 자동 검증 통과와 실제 운영 배포 완료가 구분돼 있는지

결과는 출시 차단 finding부터 심각도 순으로 작성하고, 016 세션이 반드시 재검증할 항목과 최종 판정을 제시해. 판정은 READY_FOR_FINAL_GATE 또는 BLOCKED 중 하나만 사용해. BLOCKED이면 관련 구현 세션으로 되돌릴 목록을 붙여. commit, push, deploy는 하지 마.
```
