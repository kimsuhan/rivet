# Alpha 순차 작업 프롬프트

> 이 폴더의 문서는 **사용자가 읽고 복사해 Codex에 전달하기 위한 프롬프트**다. Codex가 자동으로 적용해야 하는 작업 지침이 아니다.

## 사용 방법

1. 아래 순서대로 한 번에 하나의 프롬프트만 연다.
2. 내용을 검토하고 필요한 경우 범위나 운영 정보를 수정한다.
3. 코드 블록 안의 프롬프트 전체를 새 Codex 작업에 복사한다.
4. 작업 결과를 휴먼 리뷰하고 완료가 확인된 뒤 다음 프롬프트로 이동한다.
5. commit, push와 deploy는 각 프롬프트에서 기본적으로 금지한다. 필요할 때 사용자가 별도로 요청한다.

`007`에서 GitHub나 Slack 구현이 필요하다고 결정되면 `008`을 바로 실행하지 않는다. 실제 확인된 문제와 승인된 최소 범위에 맞는 별도 구현 프롬프트를 그때 작성해 완료한 뒤 `008`로 이동한다. 둘 다 필요하지 않다면 바로 `008`을 사용한다.

## 공통 원칙

- 각 프롬프트는 하나의 마일스톤을 끝까지 구현하고 검증하는 단위다.
- 기존 사용자 변경을 보존하고 요청 범위를 벗어난 구조 변경을 하지 않는다.
- 업무 이메일 알림은 만들지 않는다. 이메일은 회원가입·초대·이메일 인증·비밀번호 재설정에만 사용한다.
- 인앱 알림이 제품 알림의 정본이며 Web Push와 조건부 Slack은 이를 확장한다.
- 구현으로 정본 계약이 달라지면 관련 기획·아키텍처·실행 문서를 같은 작업에서 갱신한다.
- 외부 비밀, 운영 데이터와 사용자 관리 환경 파일을 읽거나 기록하지 않는다.
