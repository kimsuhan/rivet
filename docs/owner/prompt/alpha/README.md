# Alpha 순차 작업 프롬프트

> 이 폴더의 문서는 **사용자가 읽고 새 세션에 복사해 실행하기 위한 프롬프트**다. Codex나 다른 AI가 자동으로 적용해야 하는 작업 지침이 아니다.

모델 추천은 2026-07-15 현재 사용 가능한 모델과 Rivet 작업 범위를 기준으로 한다. 각 문서는 하나의 세션만 나타내며 상단에 실행할 모델 하나와 추론 강도 하나만 적는다.

## 실행 방법

1. 아래 표의 순서대로 프롬프트 하나를 새 세션에서 실행한다.
2. 구현 프롬프트가 끝나면 바로 다음 독립 검토 프롬프트를 별도 세션에서 실행한다.
3. 독립 검토가 `PASS`면 다음 번호로 이동한다.
4. 독립 검토가 `CHANGES_REQUIRED`면 검토 결과를 이전 구현 세션에 전달해 수정하고, 같은 검토 프롬프트를 다시 새 세션에서 실행한다.
5. 조건부 연동 판단은 `CONFIRMED`, 출시 사전 검토는 `READY_FOR_FINAL_GATE`일 때만 다음 단계로 이동한다.
6. commit, push와 deploy는 모든 프롬프트에서 기본적으로 금지하며 필요할 때 사용자가 별도로 요청한다.

한 문서 안에서 다른 모델에 보조 구현이나 추가 검토를 요청하지 않는다. 별도 세션이 필요한 일이 생기면 다음 단계로 넘어가기 전에 모델과 추론 강도가 하나씩 지정된 새 프롬프트 문서를 추가한다.

## 조건부 연동 분기

`014`에서 GitHub 또는 Slack 구현이 필요하다고 확정되면 `015`로 넘어가지 않는다. 실제 확인된 문제와 승인된 최소 범위를 기준으로 구현 프롬프트와 독립 검토 프롬프트를 새로 작성하고 완료한 뒤 `015`를 실행한다.

둘 다 구현하지 않는 결론이 `CONFIRMED`이면 바로 `015`로 이동한다.

## 공통 원칙

- 구현 세션은 기능 구현, 정본 문서 갱신과 전체 검증을 책임진다.
- 독립 검토 세션은 읽기 전용으로 실행하고 코드·문서를 직접 고치지 않는다.
- 독립 검토 결과는 이전 구현 세션으로 돌려보내며 검토 세션에서 수정까지 이어가지 않는다.
- 기존 사용자 변경을 보존하고 요청 범위를 벗어난 구조 변경을 하지 않는다.
- 업무 이메일 알림은 만들지 않는다. 이메일은 회원가입·초대·이메일 인증·비밀번호 재설정에만 사용한다.
- 인앱 알림이 제품 알림의 정본이며 Web Push와 조건부 Slack은 이를 확장한다.
- 외부 비밀, 운영 데이터와 사용자 관리 환경 파일을 읽거나 기록하지 않는다.
