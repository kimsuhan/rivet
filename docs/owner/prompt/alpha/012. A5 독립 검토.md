```text
Rivet Alpha A5 피드백·제품 계측 구현을 독립적으로 개인정보·관측 경계 중심으로 검토해줘. 이번 세션은 읽기 전용 검토이며 코드와 문서를 수정하지 마.

011 구현 세션의 최종 보고, Alpha 피드백·계측 요구사항, A5 실행 계획, 이벤트·관측·보안 계약, 실제 diff와 테스트를 대조해.

다음을 확인해:
- 피드백 제출·관리 권한과 workspace 격리
- 제출 실패 입력 보존, 중복 제출과 상태 변경 충돌
- 피드백 본문의 DB 보존·삭제·접근 경계
- 피드백 본문, 업무 본문, 이메일, 토큰, 파일명과 Push endpoint가 PostHog·Slack·로그로 유출되지 않는지
- 허용 이벤트 이름, payload version과 금지 필드의 강제 위치
- 외부 분석 실패가 제품 transaction·응답을 되돌리지 않는지
- Outbox 재처리와 중복이 관찰 지표를 왜곡하지 않는지
- 화면 캡처, 콘솔 로그와 첨부파일 자동 수집이 없는지
- 정의된 Alpha 관찰 지표를 실제 이벤트로 계산할 수 있는지
- API, worker, event-contracts, 관리자 UI, 정본 문서와 회귀 테스트 정합성

결과는 심각도 순 findings, 개인정보·오용 시나리오, 근거, 누락된 테스트와 최종 판정으로 작성해. 판정은 PASS 또는 CHANGES_REQUIRED 중 하나만 사용해. CHANGES_REQUIRED이면 011 구현 세션으로 돌려보낼 수정 목록을 붙여. commit, push, deploy는 하지 마.
```
