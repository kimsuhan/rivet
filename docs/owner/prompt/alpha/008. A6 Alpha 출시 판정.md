# A6 Alpha 출시 판정 프롬프트

> 사용자 열람·복사용 문서다. 자동 AI 작업 지침이 아니다.

```text
Rivet의 A6 Closed Alpha 출시 판정을 끝까지 진행해줘.

A0~A5가 완료됐고 조건부 GitHub·Slack 판단이 승인된 상태인지 먼저 확인해. 승인된 조건부 연동이 있으면 그 구현과 검증이 끝났는지도 확인하고, 끝나지 않았다면 출시 판정을 통과시키지 마. 이 요청을 하나의 Goal로 생성하고 루트 AGENTS.md, docs/index.md, Alpha 제품 범위·실행 계획, 테스트·배포 및 운영 명세, 배포 및 복구 가이드와 변경 범위에 필요한 개발 지침만 읽어.

자동 검증:
- pnpm lint
- pnpm typecheck
- pnpm test
- pnpm test:integration
- pnpm api:contract:check
- production build
- pnpm test:e2e
- Prisma format, validate, migration 상태와 빈 DB 재현성
- 생성 산출물 무차이, 문서 링크와 git diff --check

실제 제품 흐름:
- 새 사용자 가입, 인증, 워크스페이스와 팀 설정
- 관리자 CSV 업로드, 매핑, 검증, 가져오기와 실패 rollback
- 개인 저장된 보기 생성, 복원과 기본 지정
- 이슈 템플릿 관리, 적용, 수정 후 이슈 생성
- Web Push 권한, 테스트, 실제 알림, 딥 링크와 해제
- 피드백 제출과 관리자 상태 관리
- 다른 workspace와 일반 멤버의 금지 경로
- 재연결, version 충돌, 중복 재시도와 삭제·보관 대상 처리

운영 검증:
- 운영 인증 이메일 송수신은 사용자가 이미 확인한 현재 계약을 기준으로 하되 비밀이나 메일 본문을 읽지 않는다.
- migration 사전 검사, 배포·되돌리기 rehearsal
- PostHog 허용 이벤트·정제 예외와 Slack 운영 경고 smoke
- PostgreSQL·파일 저장소 백업 쌍과 격리 복구
- Push VAPID 비밀, service worker와 subscription 정리
- 로그·분석·오류에서 금지 데이터 검색

출시 차단 조건 하나라도 확인되면 실패를 숨기거나 예외 처리하지 말고 원인을 수정한 뒤 전체 관련 검증을 다시 실행해. 실제 운영 배포, DNS·TLS 변경, 운영 비밀 변경과 외부 사용자 초대는 별도 사용자 승인 없이는 하지 마.

완료 시 Alpha 제품 범위와 실행 계획의 상태, 필요한 완료 보고서와 docs/index.md의 현재 단계를 사실에 맞게 갱신해. 단, 자동 검증 통과를 실제 운영 배포 완료로 표현하지 마.

commit, push, deploy는 하지 마. 최종 보고에는 출시 판정, 기능별 인수 결과, 전체 검증 수치, 운영 준비, 남은 위험, 실제 배포 전 사용자 승인 항목과 commit/push/deploy 미실행 여부를 포함해.
```
