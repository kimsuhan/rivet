Rivet Alpha의 A3 Web Push 기능을 구현해줘.

A2 구현과 006 독립 검토가 PASS인지 먼저 확인하고 이 요청을 하나의 Goal로 생성해. 루트 AGENTS.md, docs/index.md, Alpha 범위·실행 계획의 A3, 실시간·알림 및 비동기 작업 명세와 변경 계층의 개발 지침만 읽어.

기존 인앱 알림을 유일한 알림 정본으로 유지하면서 사용자가 앱을 열어 두지 않았을 때 중요한 개인 사건을 Web Push로 받고 정확한 업무 위치로 돌아오게 해. 업무 이메일 알림은 절대 추가하지 마.

필수 범위:
- 사용자의 명시적 동작에서만 브라우저 알림 권한 요청
- 브라우저별 Push subscription 등록, 목록, 테스트 발송과 해제
- 지원, 기본, 허용, 거절, 만료 상태와 해결 방법 구분
- 인앱 notification ID와 이동 대상을 그대로 사용하는 Push 작업
- notification과 subscription 조합의 중복 전달 방지
- 만료·영구 실패 구독 비활성화와 재등록 흐름
- 할당, 멘션, 작업 전달과 구독 이슈 사건의 정확한 딥 링크
- 잠금 화면 payload에서 업무 본문, 파일명, 이메일과 민감 정보 제외
- endpoint, key material과 VAPID private key의 로그·분석·오류 노출 방지
- 로그아웃, 멤버 비활성화와 서비스 워커 갱신 정책

VAPID private key와 운영 비밀은 저장소나 데이터베이스에 넣지 말고 사용자 관리 환경으로만 주입하게 해. 외부 공급자 실패나 Push 미지원·거절이 인앱 알림 생성과 읽음 상태를 바꾸지 않게 해.

데이터 모델, worker, API, service worker, 웹 설정 UI, OpenAPI, 생성 클라이언트, 이벤트 계약, 테스트와 관련 정본 문서를 같은 작업에서 갱신해. 다중 브라우저, 중복, 만료, 권한 격리와 실제 지원 브라우저의 background 수신·클릭을 검증하고 전체 회귀를 실행해.

모바일 네이티브 푸시, 이메일 알림, Slack 개인 DM과 별도 수신자 계산은 구현하지 마. commit, push, deploy는 하지 말고 최종 보고에 알림 흐름, 비밀 관리, 변경 파일, migration, 브라우저 검증, 남은 운영 설정과 미실행 항목을 포함해.
