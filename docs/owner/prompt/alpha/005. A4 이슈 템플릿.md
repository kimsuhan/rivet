# A4 이슈 템플릿 프롬프트

> 사용자 열람·복사용 문서다. 자동 AI 작업 지침이 아니다.

```text
Rivet Alpha의 A4 이슈 템플릿 기능을 구현해줘.

A3 완료 상태를 먼저 확인하고 이 요청을 하나의 Goal로 생성해. 루트 AGENTS.md, docs/index.md, Alpha 범위·실행 계획의 A4, 이슈 생성·프로젝트 역할·디자인 관련 현행 계약과 변경 계층의 개발 지침만 읽어.

관리자가 워크스페이스 이슈 템플릿을 관리하고 사용자가 이슈 생성 화면에서 템플릿을 적용한 뒤 값을 자유롭게 수정할 수 있게 해.

필수 범위:
- 템플릿 이름, 설명 Markdown, 기본 우선순위, 라벨, 선택적 프로젝트와 최초 역할, 보관 상태와 version 저장
- 관리자 생성·수정·보관과 일반 멤버의 활성 템플릿 조회
- 적용 시 현재 라벨, 프로젝트, 역할과 팀 권한 재검증
- 템플릿 선택 전에 입력한 값이 있으면 덮어쓸 범위 확인
- 적용 뒤 이슈 입력을 자유롭게 수정하고 원본 템플릿은 변경하지 않음
- 템플릿 수정이 과거에 생성된 이슈를 바꾸지 않음
- 보관되거나 권한을 잃은 템플릿의 입력 보존과 재선택 안내
- 오래된 version이 최신 템플릿을 조용히 덮어쓰지 않게 처리
- 활동·분석 이벤트에 템플릿 본문 전체를 복제하지 않음

템플릿은 이슈 생성 입력을 채우는 수단이며 생성된 이슈와 지속적인 동기 관계를 만들지 마. 반복 일정 자동 생성, 자동화 실행, 팀별 비공개 템플릿과 외부 공유도 포함하지 마.

데이터 모델, API, OpenAPI, 생성 클라이언트, 설정·이슈 생성 UI, 키보드 접근성, 테스트와 관련 정본 문서를 함께 갱신해. 정상 적용, 보관, 권한, version 충돌과 과거 이슈 불변성을 unit, integration과 browser test로 검증하고 전체 회귀를 실행해.

commit, push, deploy는 하지 마. 최종 보고에는 관리·적용 흐름, 권한·동시성 계약, 변경 파일, migration, 검증 결과, 비목표와 미실행 항목을 포함해.
```
