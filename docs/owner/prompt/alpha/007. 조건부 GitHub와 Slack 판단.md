# 조건부 GitHub·Slack 판단 프롬프트

> 사용자 열람·복사용 문서다. 자동 AI 작업 지침이 아니다.

```text
Rivet Alpha의 조건부 GitHub·Slack 제품 연동 필요성을 판단해줘. 이번 작업은 분석과 추천만 수행하고 연동 코드는 구현하지 마.

A5까지 완료됐는지 확인하고 루트 AGENTS.md, docs/index.md, Alpha 제품 범위의 조건부 연동 기준, Alpha 실행 계획의 11절과 현재 확보된 피드백·제품 이벤트 근거만 읽어. 사용자 데이터, 운영 비밀과 외부 계정에 임의로 접근하지 마.

다음 세 선택지를 같은 기준으로 비교해:
1. GitHub 제한 연동을 Alpha에 추가
2. Slack 제한 연동을 Alpha에 추가
3. 둘 다 구현하지 않고 A6로 진행

반드시 확인할 근거:
- 같은 문제가 발생한 workspace와 사용자 수
- 문제의 반복 빈도와 현재 우회 방법의 비용
- 첫 범위로 실제 줄어드는 조작과 사용자 영향
- 필요한 외부 권한, 비밀, webhook, rate limit과 장애 대응
- 데이터 보존·삭제·감사와 공급자 장애 영향
- 구현하지 않을 때 Alpha·Beta에 미치는 영향
- A6를 늦출 만큼 중요한지 여부

근거가 부족하면 추측으로 연동을 추천하지 말고 둘 다 미구현을 추천해. GitHub를 선택하면 PR 링크와 상태의 읽기 표현부터, Slack을 선택하면 관리자가 등록한 Incoming Webhook과 소수의 팀·프로젝트 공용 사건부터 검토해. 양방향 댓글·명령, 자동 상태 전이, 개인 DM과 두 연동의 동시 구현은 추천하지 마.

결과는 결론을 먼저 제시하고 근거, 반대 근거, 최소 범위, 운영 비용, 비목표와 Beta 재검토 조건을 포함한 의사결정 보고서로 작성해. 코드와 정본 문서는 수정하지 말고 내 승인을 기다려. commit, push, deploy도 하지 마.
```
