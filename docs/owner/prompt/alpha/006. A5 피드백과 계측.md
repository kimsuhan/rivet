# A5 피드백·계측 프롬프트

> 사용자 열람·복사용 문서다. 자동 AI 작업 지침이 아니다.

```text
Rivet Alpha의 A5 피드백·제품 계측 기능을 구현해줘.

A1~A4 완료 상태를 먼저 확인하고 이 요청을 하나의 Goal로 생성해. 루트 AGENTS.md, docs/index.md, Alpha 범위·실행 계획의 A5, 현재 이벤트·관측·보안 계약과 변경 계층의 개발 지침만 읽어.

인증된 사용자가 제품 안에서 정성 피드백을 제출하고, 운영자가 Alpha의 핵심 사용 흐름을 민감 정보 없이 관찰할 수 있게 해.

피드백 필수 범위:
- 카테고리, 본문, 제출 membership·workspace, 현재 경로, release ID, 상태와 시각 저장
- 인증 사용자 제출과 관리자 목록·상태 변경
- 접수, 검토 중, 반영, 보류 상태 관리
- 제출 성공 확인, 실패 시 입력 보존과 중복 제출 방지
- 피드백 본문을 PostHog, Slack 운영 경고와 일반 로그에 보내지 않음
- 화면 캡처, 콘솔 로그, 이슈·댓글·전달 본문과 첨부파일 자동 수집 금지
- workspace 격리, 보존과 삭제 영향 검증

제품 이벤트 필수 범위:
- 허용 이벤트와 payload 버전을 event-contracts에 정의
- 초대·가입, CSV 가져오기, 첫 이슈·작업 시작·완료, 저장된 보기, 템플릿, Push, 검색, 알림 읽음과 피드백 제출 사건
- 업무 본문, 이메일, 토큰, 파일명, Push endpoint와 피드백 본문을 payload에서 거부
- 외부 분석 전송 실패가 원래 제품 트랜잭션과 성공 응답을 되돌리지 않음
- 중복·재처리에도 지표가 왜곡되지 않는 식별 계약

Alpha 표본이 작으므로 임의의 성공률 목표나 분석 대시보드를 만들지 마. 문서에 정의된 관찰 지표를 실제 이벤트로 계산할 수 있는지 샘플 데이터로 대조하는 데 집중해.

데이터 모델, API, worker·Outbox, 이벤트 계약, 웹 UI, 관리자 화면, 테스트와 관련 정본 문서를 같은 작업에서 갱신해. 허용·금지 필드, workspace 권한, 외부 공급자 실패와 중복 처리를 unit, integration과 browser test로 검증하고 전체 회귀를 실행해.

commit, push, deploy는 하지 마. 최종 보고에는 피드백 흐름, 이벤트 목록과 금지 데이터, 변경 파일, migration, 검증 결과, 실제 관찰 가능 지표와 미실행 항목을 포함해.
```
