# A1 독립 검토 프롬프트

> 사용자 열람·복사용 문서다. `003` 구현 세션과 분리된 새 세션에서 실행한다.

## 실행 설정

| 항목 | 추천 |
| --- | --- |
| 모델 | Claude Opus |
| 추론 강도 | high |

```text
Rivet Alpha A1 CSV 가져오기 구현을 독립적으로 검토해줘. 이번 세션은 읽기 전용 검토이며 코드와 문서를 수정하지 마.

003 구현 세션의 최종 보고, Alpha CSV 요구사항, A1 실행 계획, 실제 diff와 테스트를 대조해. 구현 세션이 주장한 성공 여부와 관계없이 데이터 안전성, 권한과 실패 복구를 우선 검증해.

다음을 확인해:
- 허용 형식·인코딩·크기·행 수·필수 컬럼과 formula injection 방어
- 팀·상태·멤버·프로젝트·우선순위·라벨 매핑의 workspace 격리
- preview와 실제 저장 사이 대상 변경·version 충돌 처리
- 전체 성공 또는 전체 rollback의 transaction 경계
- source key·실행 식별자 기반 중복 방지와 동시 실행
- 비관리자, 다른 workspace ID와 보관·비활성 대상 거부
- 원본 CSV·업무 본문·민감 값의 저장·로그·분석 노출 여부
- 미지원 댓글·활동·전달·첨부파일을 조용히 버리지 않는지
- migration, OpenAPI, 생성 클라이언트, UI와 정본 문서 정합성
- 정상·실패·경계·권한 browser와 PostgreSQL 통합 테스트의 충분성

결과는 심각도 순 findings, 재현 근거, 누락된 테스트와 최종 판정으로 작성해. 판정은 PASS 또는 CHANGES_REQUIRED 중 하나만 사용해. CHANGES_REQUIRED이면 003 구현 세션으로 돌려보낼 구체적 수정 목록을 붙여. commit, push, deploy는 하지 마.
```
