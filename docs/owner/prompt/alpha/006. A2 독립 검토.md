# A2 독립 검토 프롬프트

> 사용자 열람·복사용 문서다. `005` 구현 세션과 분리된 새 세션에서 실행한다.

## 실행 설정

| 항목 | 추천 |
| --- | --- |
| 모델 | Claude Sonnet |
| 추론 강도 | high |

```text
Rivet Alpha A2 개인 저장된 보기 구현을 독립적으로 검토해줘. 이번 세션은 읽기 전용 검토이며 코드와 문서를 수정하지 마.

005 구현 세션의 최종 보고, Alpha 저장된 보기 요구사항, A2 실행 계획, 실제 diff와 테스트를 대조해.

다음을 확인해:
- membership 소유권과 workspace 직접 접근 차단
- 허용된 검색·필터·정렬·표시 값만 저장하는지
- 저장·열기·이름 변경·삭제·기본 지정의 정상·충돌 흐름
- 리소스 종류별 기본 보기 하나를 보장하는 transaction
- 저장된 보기, URL과 실제 화면 상태의 왕복 일치
- 권한을 잃거나 보관된 값에 대한 명시적 복구 안내
- 이름 충돌, 오래된 version과 동시 수정 처리
- pagination, 정렬 안정성, 직접 진입, 새로고침, 뒤로 가기와 스크롤 복원 회귀
- API, 생성 클라이언트, UI, 접근성과 관련 정본 문서 정합성
- 개인·workspace 격리와 browser test의 충분성

결과는 심각도 순 findings, 근거, 누락된 테스트와 최종 판정으로 작성해. 판정은 PASS 또는 CHANGES_REQUIRED 중 하나만 사용해. CHANGES_REQUIRED이면 005 구현 세션으로 돌려보낼 수정 목록을 붙여. commit, push, deploy는 하지 마.
```
