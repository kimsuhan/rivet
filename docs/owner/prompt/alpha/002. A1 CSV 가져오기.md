# A1 CSV 가져오기 프롬프트

> 사용자 열람·복사용 문서다. 자동 AI 작업 지침이 아니다.

```text
Rivet Alpha의 A1 CSV 가져오기 기능을 제품 흐름 전체로 구현해줘.

A0 완료 상태를 먼저 검증하고 이 요청을 하나의 Goal로 생성해. 루트 AGENTS.md, docs/index.md, docs/planning/009. Alpha 제품 범위 및 검증 계획서.md의 CSV 요구사항, docs/delivery/005. Alpha 개발 실행 계획서.md의 A1과 변경 계층에 필요한 현행 계약·개발 지침만 읽어.

관리자가 CSV를 선택하고 컬럼과 값을 매핑한 뒤 저장 전 검증 결과를 확인하고, 부분 저장 없이 프로젝트와 이슈를 가져오는 흐름을 완성해. 파일 선택 → 컬럼 확인 → 값 매핑 → 검증 → 최종 확인 → 결과까지 실제 UI와 API로 연결해.

필수 범위:
- 허용 형식, 인코딩, 파일 크기, 행 수와 필수 컬럼 검증
- 팀, 상태, 멤버, 프로젝트, 우선순위와 라벨 매핑
- 생성·연결·제외 예정 수와 행별 오류 preview
- 전체 성공 또는 전체 rollback의 원자적 저장
- source key와 실행 식별자 기반 중복 실행 방지
- 관리자 권한과 workspace 격리
- 실행 결과와 정제된 오류 추적
- 원본 CSV와 업무 본문을 장기 저장하거나 로그·분석에 노출하지 않는 정책
- 댓글, 활동, 알림, 전달 이력, 첨부파일과 외부 자동화 규칙을 조용히 버리지 않고 미지원으로 안내

정상 CSV뿐 아니라 빈 파일, 잘못된 인코딩·헤더·행, formula injection, 최대 경계, 보관된 대상, 알 수 없는 매핑 값, 중복과 동시 실행을 검증해. 실패 뒤 일부 데이터가 남지 않는지 실제 PostgreSQL 통합 테스트로 증명하고 다른 workspace의 매핑·결과 접근을 거부해.

Prisma schema, migration, API, OpenAPI, 생성 클라이언트, 웹 UI, 테스트와 관련 정본 문서를 같은 작업에서 완성해. 정상 구현 뒤 관련 unit, integration, contract, browser 테스트와 전체 회귀를 실행해.

범위를 외부 시스템 지속 동기화나 범용 importer로 확대하지 마. commit, push, deploy는 하지 말고 최종 보고에 사용자 흐름, 데이터·권한 계약, 변경 파일, migration, 검증 결과, 비목표와 미실행 항목을 포함해.
```
