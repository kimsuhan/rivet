```text
Rivet Alpha A3 Web Push 구현을 독립적으로 보안·정합성 검토해줘. 이번 세션은 읽기 전용 검토이며 코드와 문서를 수정하지 마.

007 구현 세션의 최종 보고, Alpha Web Push 요구사항, A3 실행 계획, 실시간·알림 명세, 실제 diff와 테스트를 대조해. 인앱 알림이 유일한 정본이라는 계약을 기준으로 검토해.

다음을 확인해:
- 사용자 동작 없이 브라우저 권한을 요청하지 않는지
- subscription 소유권, membership·workspace 격리와 다중 브라우저 처리
- notification과 subscription 조합의 멱등성·중복·폭주 방지
- 만료·영구 실패, 로그아웃과 멤버 비활성화 처리
- 인앱 notification ID·수신자·딥 링크를 재사용하고 별도 정본을 만들지 않는지
- 업무 본문·파일명·이메일이 Push payload와 잠금 화면에 노출되지 않는지
- endpoint, key material과 VAPID private key가 DB 외 허용 경계, 로그·분석·오류에서 보호되는지
- service worker 갱신과 Push 실패가 앱 로딩·인앱 알림을 막지 않는지
- 실제 지원 브라우저 background 수신·클릭 검증과 전체 회귀
- 업무 이메일이나 Slack 개인 알림이 범위에 섞이지 않았는지

결과는 심각도 순 findings, 공격·실패 시나리오, 근거, 누락된 테스트와 최종 판정으로 작성해. 판정은 PASS 또는 CHANGES_REQUIRED 중 하나만 사용해. CHANGES_REQUIRED이면 007 구현 세션으로 돌려보낼 수정 목록을 붙여. commit, push, deploy는 하지 마.
```
