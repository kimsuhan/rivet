# A2 개인 저장된 보기 프롬프트

> 사용자 열람·복사용 문서다. 자동 AI 작업 지침이 아니다.

```text
Rivet Alpha의 A2 개인 저장된 보기 기능을 구현해줘.

A1 완료 상태를 먼저 확인하고 이 요청을 하나의 Goal로 생성해. 루트 AGENTS.md, docs/index.md, Alpha 범위·실행 계획의 A2, 목록 관련 현행 제품 계약과 변경 계층의 개발 지침만 읽어.

전역 이슈와 내 작업에서 사용자가 현재 검색, 빠른 필터, 세부 필터, 정렬과 표시 옵션을 이름 붙여 저장하고 다시 열며 기본 보기로 지정할 수 있게 해. 개인 보기이므로 다른 멤버와 공유하지 않는다.

필수 범위:
- membership 소유권, workspace, 리소스 종류, 이름, 정규화된 query configuration, 기본 여부와 version 저장
- 허용된 필터·정렬·표시 키만 저장하고 임의 SQL·URL·미지원 속성 거부
- 목록, 생성, 이름 변경, 삭제와 기본 지정 API
- 리소스 종류별 기본 보기 하나를 트랜잭션으로 보장
- 저장된 보기 선택과 현재 URL·화면 상태의 일치
- 권한을 잃거나 보관된 값에 대한 명시적 수정 안내
- 이름 충돌, 오래된 version, 동시 수정과 삭제 실패 복구
- 다른 membership과 workspace의 직접 조회·수정 거부

저장된 보기를 열어도 기존 pagination, 정렬 안정성, URL 직접 진입, 새로고침, 뒤로 가기와 목록 스크롤 복원 계약이 깨지지 않게 해. 팀 공유 보기, 구독, 권한별 공유와 복잡한 중첩 OR 필터는 구현하지 마.

데이터 모델, API, OpenAPI, 생성 클라이언트, 웹 UI, 접근성, 테스트와 관련 정본 문서를 함께 갱신해. 정상·충돌·권한·보관 값 흐름을 unit, PostgreSQL integration과 실제 browser test로 검증하고 전체 회귀를 실행해.

commit, push, deploy는 하지 마. 최종 보고에는 저장·복원 흐름, 소유권 계약, 변경 파일, migration, 검증 결과, 비목표와 미실행 항목을 포함해.
```
