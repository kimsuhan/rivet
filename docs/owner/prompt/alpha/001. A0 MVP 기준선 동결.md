# A0 MVP 기준선 동결 프롬프트

> 사용자 열람·복사용 문서다. 자동 AI 작업 지침이 아니다.

```text
Rivet Alpha의 A0 MVP 기준선 동결 작업을 진행해줘.

이 요청을 하나의 Goal로 생성하고, 아래 완료 조건을 모두 충족할 때까지 범위 안에서 자율적으로 진행해. 작업 시작 전 git status, 루트 AGENTS.md, docs/index.md, docs/delivery/004. MVP 개발 완료 보고서.md, docs/delivery/005. Alpha 개발 실행 계획서.md와 변경 경로에 필요한 개발 지침만 읽어.

작업 범위:
1. M9 E2E의 담당자 선택기 locator를 현재 combobox 역할과 접근 가능한 이름에 맞춘다.
2. E2E가 소스 Prisma Client를 읽는 TypeScript 실행 계약을 임시 NODE_OPTIONS 없이 저장소 명령으로 재현 가능하게 만든다.
3. 사용자가 실행 중인 기본 포트와 충돌하지 않는 격리 포트 실행 경로를 만든다. 기존 프로세스를 임의로 종료하지 않는다.
4. desktop, compact, mobile Chromium 전체 회귀를 다시 실행한다.
5. 로컬 rivet 데이터베이스의 public 스키마에 한정해 빈 스키마부터 전체 migration 재적용, schema diff와 통합 테스트를 검증한다.
6. lint, typecheck, unit, integration, API contract와 production build를 같은 HEAD에서 재검증한다.
7. 검증 결과가 달라지면 MVP 완료 보고서와 Alpha 실행 계획의 A0 상태를 현재 근거에 맞게 갱신한다.

이 프롬프트는 로컬 rivet 데이터베이스의 테스트 전용 public 스키마 reset만 허용한다. 다른 데이터베이스나 스키마는 절대 초기화하지 마. Prisma가 경고 표시 이후의 별도 사용자 동의를 다시 요구하면 우회하지 말고 그 시점에만 나에게 정확한 동의를 요청해.

완료 조건:
- 제품 E2E 실패 0건이고 viewport 조건에 따른 의도적 skip만 남는다.
- 빈 테스트 스키마에서 migration 전체가 순서대로 적용되고 schema diff가 없다.
- 모든 필수 검증이 통과한다.
- 임시 환경 변수나 개인 로컬 파일에 의존하지 않는 재현 가능한 명령이 저장소에 남는다.

실패한 검증은 숨기거나 제외하지 말고 원인과 남은 위험을 해결해. 코드와 문서 변경을 모두 휴먼 리뷰할 수 있게 정리하고 commit, push, deploy는 하지 마. 최종 보고에는 변경 파일, 재현 명령, 검증 결과, 남은 항목과 commit/push/deploy 미실행 여부를 포함해.
```
