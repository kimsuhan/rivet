# 문서 색인

Rivet의 현재 개발 단계는 **Closed Alpha**다. 기본 작업 입력은 Alpha 범위와 실행 계획이며, 완료된 MVP 기획·실행 문서는 과거 판단이나 회귀 기준을 확인할 때만 읽는다.

문서를 번호순으로 전부 읽지 않는다. 현재 작업에 필요한 제품 계약, 기술 계층과 개발 지침만 선택한다.

## 현재 단계: Closed Alpha

Alpha 기능을 기획하거나 구현할 때는 다음 두 문서에서 시작한다.

| 문서 | 사용하는 경우 | 상태 |
| --- | --- | --- |
| [Alpha 제품 범위 및 검증 계획서](./planning/009.%20Alpha%20제품%20범위%20및%20검증%20계획서.md) | Alpha 목표, 우선순위, 인수 조건, 비목표와 Beta 전환 기준을 판단할 때 | 확정 v1.2 |
| [Alpha 개발 실행 계획서](./delivery/005.%20Alpha%20개발%20실행%20계획서.md) | A0~A6 작업 순서, 계층별 구현 범위, 검증과 출시 차단 조건을 확인할 때 | 확정 v1.11 — A0·A1·A2·A3 독립 검토 PASS |

Alpha의 확정 우선순위는 CSV 가져오기, 개인 저장된 보기, Web Push, 이슈 템플릿, 피드백·계측 순서다. GitHub 또는 Slack 제품 연동은 실제 반복 문제가 확인될 때만 별도 범위로 추가한다. 업무 이메일 알림은 Alpha, Beta와 Public 범위에 포함하지 않는다.

## 작업별 문서 선택

### 새 Alpha 기능

1. [Alpha 제품 범위 및 검증 계획서](./planning/009.%20Alpha%20제품%20범위%20및%20검증%20계획서.md)에서 요구사항과 비목표를 확인한다.
2. [Alpha 개발 실행 계획서](./delivery/005.%20Alpha%20개발%20실행%20계획서.md)에서 해당 마일스톤과 검증 범위를 확인한다.
3. 아래 현행 제품·기술 계약에서 변경되는 문서만 읽고 같은 작업에서 갱신한다.
4. 코드 작업이면 [개발 지침 색인](./development/index.md)에서 변경 범위에 지정된 지침만 추가로 읽는다.

### 기존 기능 수정 또는 회귀 대응

- Alpha 문서 전체를 다시 읽지 않고 아래 현행 제품·기술 계약에서 변경되는 영역만 선택한다.
- 기존 동작을 바꾸는 제품 결정이면 Alpha 범위 문서와 관련 현행 계약을 함께 갱신한다.
- A0 이후 MVP 회귀 근거나 동결된 기준선을 확인할 때만 [MVP 개발 완료 보고서](./delivery/004.%20MVP%20개발%20완료%20보고서.md)를 읽는다.

### 배포·복구·운영

- 최초 배포, 되돌리기, 백업과 복구 실행은 [비공개 릴리스 배포 및 복구 가이드](./delivery/003.%20비공개%20베타%20배포%20및%20복구%20가이드.md)를 사용한다.
- 이미 배포된 서버에 새 릴리스만 반영할 때는 [버전 업데이트 배포 가이드](./delivery/006.%20버전%20업데이트%20배포%20가이드.md)의 반복 절차와 흔한 실패 체크리스트를 사용한다.
- 테스트 계층, migration, PM2·Nginx와 관측 기준은 [테스트·배포 및 운영 명세서](./architecture/006.%20테스트%C2%B7배포%20및%20운영%20명세서.md)를 함께 확인한다.
- 운영 인증 이메일은 현재 계약을 유지하며 제품 알림 채널로 확장하지 않는다.

## 문서 관리 규칙

- 현재 단계의 기본 진입점은 이 색인 상단에 두고 완료된 단계 문서는 `완료된 이력`으로 내린다.
- 완료 이력은 삭제하거나 현재 요구사항처럼 소급 수정하지 않는다. 현재 제품 정책이 달라지면 Alpha 이후 문서와 현행 계약을 수정한다.
- `planning/`은 제품 범위와 사용자 동작, `architecture/`는 현재 기술 계약, `delivery/`는 실행·검증·운영 기록을 관리한다.
- 코드 작업용 `development/` 문서는 전체를 읽지 않고 [개발 지침 색인](./development/index.md)에서 지정한 범위만 읽는다.
- 새 문서는 의존 순서에 따라 `001. 문서명.md` 형식으로 번호를 붙이고 이 색인에 현재 역할과 상태를 추가한다.
- 문서 링크, 상태나 역할이 바뀌면 이 색인과 관련 본문을 같은 작업에서 갱신한다.
- 검토 과정에서 확정한 값은 해당 정본 본문에 직접 반영하고 별도 결정 요약을 중복해서 만들지 않는다.
