# 보안 운영과 외부 연동

> 상태: 현행
> 적용 범위: 운영 비밀, 보안 사고, Resend, Web Push, PostHog와 Slack
> 정본 책임: 공급자 설정·점검과 보안 대응 절차
> 갱신 조건: 비밀, 외부 공급자, 관측 채널 또는 사고 대응 방식 변경

## 비밀 운영

- 운영 비밀은 저장소와 릴리스 디렉터리 밖에 보관한다.
- 공급자와 환경별로 키를 분리하고 최소 권한을 사용한다.
- 키, webhook URL, VAPID private key와 HMAC 원문을 로그·PM2 dump·배포 기록에 남기지 않는다.
- PM2 환경을 확인할 때 `pm2 env`나 `jlist` 전체 출력을 작업 기록에 첨부하지 않는다.
- 교체 뒤에는 프로세스를 새로 생성해 제거한 이전 값이 남지 않게 한다.

## 비밀 교체

1. 새 비밀을 공급자에서 발급하고 대상 환경·프로젝트를 확인한다.
2. 권한이 제한된 운영 환경 저장소에 새 값을 반영한다.
3. 영향받는 프로세스를 `pm2 delete` 후 새 환경으로 시작한다.
4. health와 해당 공급자 smoke test를 수행한다.
5. 새 값 사용을 확인한 뒤 이전 비밀을 폐기한다.
6. 값이 아닌 교체 시각, 대상, 실행자와 검증 결과만 기록한다.

HMAC 키 교체는 기존 세션·토큰·rate limit 식별자에 미치는 영향을 별도로 검토한다. 데이터
호환 전략 없이 즉시 교체하지 않는다.

## 보안 사고

다른 워크스페이스 노출, 권한 우회와 세션·토큰·비밀 노출은 긴급 사고다.

1. 관련 엔드포인트, 외부 연동 또는 전체 쓰기를 중단한다.
2. 노출된 세션, 토큰과 공급자 비밀을 폐기한다.
3. 로그와 관측 데이터에서 노출 범위를 확인하되 민감 값을 추가 복제하지 않는다.
4. 최근 배포·설정·권한 변경과 영향받은 워크스페이스 범위를 확인한다.
5. 수정 뒤 격리, 권한, CSRF, 토큰 재사용과 민감 로그 회귀 테스트를 실행한다.
6. 사용자 영향, 복구와 후속 조치를 승인된 사고 기록에 남긴다.

자동 테스트와 실제 운영 확인이 끝나기 전 쓰기를 재개하지 않는다.

## Resend

- 검증된 발신 도메인과 주소를 사용한다.
- 개발·테스트는 `RESEND_ALLOWED_RECIPIENTS`로 수신자를 제한한다.
- 계정 확인, 재설정과 초대 외 제품 알림을 추가하지 않는다.
- 공급자 응답 본문과 이메일 주소를 일반 로그에 남기지 않는다.
- 실제 수신 점검은 전용 계정과 만료 가능한 테스트 토큰으로 수행한다.

## Web Push

- 공개 키는 API와 Worker가 공유하고 private key와 subject는 Worker에만 둔다.
- 브라우저 권한은 사용자의 설정 동작에서만 요청한다.
- 시험 Push는 사용자가 선택한 자신의 활성 구독으로만 보낸다.
- 만료·폐기 세션과 endpoint는 비활성화하고 endpoint·키를 제거한다.
- Push payload에 이슈·댓글·전달 본문과 사용자 표시 값을 넣지 않는다.

## PostHog와 Slack

- PostHog에는 허용된 제품 이벤트와 정제한 예외만 전송한다.
- Slack은 긴급·높음 수준의 운영 경고에만 사용한다.
- 시험 프로젝트·채널과 운영 프로젝트·채널을 분리한다.
- 실제 전송 전 dry-run으로 대상 이름과 payload 종류를 확인한다.
- 운영 smoke는 명시적인 확인 문자열 없이는 외부 호출하지 않는다.

```sh
pnpm observability:smoke -- --dry-run
```

실제 전송 절차와 확인 문자열은
[`scripts/check-external-observability.mjs`](../../scripts/check-external-observability.mjs)의
현재 CLI 계약을 확인한 뒤 사용한다. 비밀 값과 공급자 응답 원문을 명령 출력에 포함하지 않는다.

## 외부 연동 추가 기준

새 외부 연동은 다음을 구현 전에 확정한다.

- 실제 사용자 문제와 최소 권한
- 설치·연결 해제·키 교체 책임자
- 요청 서명 또는 OAuth 상태 검증
- webhook 멱등 키와 빠른 응답 경계
- payload 최소화와 보존 기간
- 누락·순서 역전·재전송 복구
- 공급자 장애가 제품 원본 상태에 미치는 영향
- 재동기화, 관측, 롤백과 데이터 삭제 절차

기존에 작성된 미구현 연동 계획은 현재 제품 계약이 아니다. Public 범위에서 다시 승인한
연동만 새 실행 문서와 아키텍처 변경으로 추가한다.
